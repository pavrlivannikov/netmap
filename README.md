# NetMap v1.0.13 — Полная стратегия использования

> Десктопный сетевой сканер. Portable exe, без зависимостей. Windows/Linux.

---

## 1. Начало работы

### Запуск
- **Windows:** `netmap-v1.0.13.exe` (портативный, не требует установки, без админ-прав)
- **Linux:** `python3 netmap_gui.py` (требуется: `python3-tk`, `netifaces`)

### Первый запуск
1. Программа автоматически обнаружит активную сеть (выпадающий список вверху)
2. Если сетей несколько — выберите нужную
3. Кнопка 🔄 — обновить список сетей

---

## 2. Режимы сканирования

| Кнопка | Режим | Время | Что делает |
|--------|-------|-------|------------|
| ⚡ Быстрый | ARP | < 5 сек | Только ARP-таблица: IP, MAC, vendor |
| 🔍 Обзор | ARP + ICMP + TCP | 1-3 мин | ARP + пинг-свип + порты + OS по TTL |
| 🕳 Глубокий | всё + nmap | 5-15 мин | Обзор + nmap OS detection (нужен nmap в PATH) |
| 🔗 Топология | всё + SNMP | 1-5 мин | Обзор + SNMP (LLDP-связи + FDB/MAC-таблица) |

### Рекомендуемая стратегия
1. **Первый раз:** Быстрый → базовая картина, кто в сети
2. **Детальный анализ:** Топология → SNMP на коммутаторах/роутерах даст связи
3. **Периодический контроль:** Монитор (авто-повтор Быстрого)

## 3. Вкладки

### 📋 Устройства (основная таблица)
- **Колонка 🔔** — отметить устройство для оповещения (ПКМ → Вкл/выкл оповещение)
- Сортировка по любой колонке (клик по заголовку)
- **ПКМ** на строке: копировать IP/MAC, сканировать порты, оповещение
- 🟢 зелёный = online, 🔴 красный = пропало (мониторинг), зелёный фон = появилось

### 📝 Топология (текст)
- Группировка по типам устройств
- Список связей (edges) с портами и VLAN
- **Ctrl+C / ПКМ** — копировать текст

### 🕸 Граф (интерактивная карта)
- **ЛКМ** на узле — перетащить
- **Ctrl/Alt + ЛКМ** — панорама
- **Колёсико** — зум
- **Стрелки** на клавиатуре — навигация
- **Мини-карта** (правый нижний угол) — общий вид, клик для перемещения
- **Ховер** на узле — подсказка с IP, MAC, портами
- Кнопки: **🔍 Уместить всё** — сбросить зум, **📐 Авто-раскладка** — упорядочить

### ⚙ Настройки
- **Интервал мониторинга** (10-3600 сек)
- **Звук при изменениях** (вкл/выкл)
- **Список устройств с 🔔** — управление оповещаемыми
- Кнопка «Сохранить настройки»

### 📝 Лог
- Хронология всех действий
- **Ctrl+C / ПКМ** — копировать

### 📄 JSON
- Полный результат сканирования в JSON
- Для скриптов, экспорта, внешней обработки

## 4. Мониторинг

### Запуск
1. Выполните **Быстрый** скан (базовый снимок сети)
2. Нажмите **📊 Монитор** → кнопка сменится на ⏹ Стоп монитор
3. В статус-баре появится 🟢 Мониторинг (60с)

### Как работает
- Каждые N секунд (настраивается в ⚙) — автоматический Быстрый скан
- Сравнение с предыдущим снимком
- При изменениях:
  - **Звук** (если включён)
  - **Цветной diff** в таблице (🟢 появилось / 🔴 пропало)
  - **Всплывающее окно** — для отмеченных 🔔 устройств

### Стратегия мониторинга
1. Быстрый скан → отметить критичные устройства (ПКМ → 🔔)
2. В ⚙ задать интервал (30-60 сек для оперативного, 5-10 мин для фонового)
3. Запустить Монитор → следить за статус-баром
4. При пропадании важного устройства — всплывёт окно

## 5. Типовые сценарии

### Аудит сети
```
⚡ Быстрый → смотрим кто есть → 🔗 Топология → полная карта с LLDP
```

### Поиск нового устройства
```
⚡ Быстрый → запомнили → через час ещё раз → 📊 Монитор → 🟢 появилось
```

### Расследование проблем
```

```

### Инвентаризация
```
🔗 Топология → JSON → экспорт → внешняя обработка
```

## 6. SNMP (режим Топология)

- Community: **public** (стандартный)
- Таймаут: **0.5 сек** на устройство
- Что собирается:
  - **sysName** → hostname
  - **sysDescr** → OS
  - **LLDP** (1.0.8802.1.1.2.1.4.1.1) → прямые связи между устройствами
  - **FDB** (1.3.6.1.2.1.17.7.1.2.2.1) → MAC-таблица (какой MAC на каком порту)
- Если SNMP не отвечает — fallback на TCP-порты

### SNMP не работает?
- Проверьте, что community совпадает (по умолчанию public)
- Проверьте файрвол (UDP 161)
- Устройство может не поддерживать SNMP (тогда просто TCP-порты)

## 8. Экспорт данных

### JSON файл
- 💾 Экспорт → сохраняет результат в .json
- 📂 Загрузить → открывает предыдущий скан
- Вкладка JSON → копировать raw данные

### Для внешних систем
```python
import json
with open('netmap-20260526.json') as f:
    data = json.load(f)
for device in data['devices']:
    print(f"{device['ip']} | {device['mac']} | {device['vendor']}")
```

## 9. OUI-база (определение вендора)

- **39 461 записей** из официальной базы IEEE
- Определяет производителя по MAC-адресу: Cisco, HP, Dell, Intel, Apple, Xiaomi, Huawei...
- Если вендор не определён — показан диапазон (IEEE, Samsung/Huawei)

---

## 10. Ограничения

- **ARP** — требуется, чтобы устройство было в ARP-таблице ОС
- **ICMP** — может быть заблокирован фаерволом
- **SNMP** — только v2c, community public, таймаут 0.5с
- **nmap** — для Глубокого режима должен быть установлен отдельно
- **Права** — админ-права НЕ требуются (кроме nmap на Windows для SYN-скана)
