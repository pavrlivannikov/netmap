# NetMap v1.0.17 — Десктопный сетевой сканер

> Portable. Windows/Linux. Без установки, без админ-прав.

---

## 1. Начало работы

### Запуск
- **Windows:** `netmap.exe` (портативный, один файл)
- **Linux:** `python3 netmap_gui.py` (требуется: `python3-tk`, `netifaces`, `pysnmp`)

### Первый запуск
1. Программа автоматически обнаружит активную сеть (выпадающий список вверху)
2. Если сетей несколько — выберите нужную
3. Кнопка 🔄 — обновить список сетей

---

## 2. Режимы сканирования

| Режим | Время | Что делает |
|-------|-------|------------|
| ⚡ Быстрый | < 5 сек | ARP-таблица: IP, MAC, vendor |
| 🔍 Обзор | 1-3 мин | ARP + пинг-свип + TCP-порты + OS по TTL |
| 🕳 Глубокий | 5-15 мин | Обзор + nmap OS detection (нужен nmap в PATH) |
| 🔗 Топология | 1-5 мин | Обзор + SNMP (LLDP-связи + FDB/MAC-таблица) |
| 📊 Монитор | цикл | Авто-повтор Быстрого с diff и алертами |

### Стратегия
1. **Первый раз:** Быстрый → базовая картина
2. **Детально:** Топология → SNMP на коммутаторах/роутерах
3. **Контроль:** Монитор → авто-повтор

## 3. Вкладки

### 📋 Устройства
- Таблица: 🔔, IP, MAC, hostname, vendor, тип, OS, порты, статус
- ПКМ на строке: копировать IP/MAC, сканировать порты, вкл/выкл оповещение
- 🟢 online / 🔴 пропало / зелёный фон = появилось
- Сортировка по любой колонке

### 🗂 Структура
- Древовидная иерархия: сеть → подсеть → устройство → порты/сервисы
- Ветки сворачиваются/разворачиваются

### 📝 Топология (текст)
- Группировка по типам устройств
- Список связей (edges) с портами и VLAN
- Ctrl+C / ПКМ — копировать

### 🕸 Граф
- Drag узлов (ЛКМ)
- Zoom (колёсико / кнопки 🔍− 🔍+)
- Панорама (Ctrl+ЛКМ / стрелки клавиатуры)
- Мини-карта (правый нижний угол)
- Hover на узле → подсказка (IP, MAC, порты, OS)
- 🔍 Уместить всё — сброс зума
- 📐 Авто-раскладка — упорядочить

### ⚙ Настройки
- Интервал мониторинга (10-3600 сек)
- Звук при изменениях (вкл/выкл)
- SNMP community (по умолчанию public)
- Настройки сохраняются между запусками

### 📝 Лог
- Хронология всех действий
- Ctrl+C / ПКМ — копировать

### 📄 JSON
- Полный результат сканирования в JSON
- Для скриптов, экспорта, внешней обработки

## 4. Мониторинг

1. Быстрый скан (базовый снимок)
2. Отметить критичные устройства (ПКМ → 🔔)
3. Задать интервал в ⚙ (30-60 сек для оперативного, 5-10 мин для фонового)
4. 📊 Монитор → кнопка сменится на ⏹ Стоп

При изменениях:
- 🟢 появилось / 🔴 пропало в таблице
- Звук (если включён)
- Всплывающее окно для отмеченных 🔔 устройств

## 6. SNMP (режим Топология)

- **Версия:** SNMPv2c
- **Community:** настраиваемый (по умолчанию public)
- **Таймаут:** 0.5 сек на устройство

Что собирается:
- **sysName** → hostname
- **sysDescr** → OS
- **LLDP** (1.0.8802.1.1.2.1.4.1.1) → прямые связи между устройствами
- **FDB** (1.3.6.1.2.1.17.7.1.2.2.1) → MAC-таблица (какой MAC на каком порту)

Если SNMP не отвечает → fallback на TCP-порты.

**Не работает?**
- Проверьте community
- Проверьте файрвол (UDP 161)
- Устройство может не поддерживать SNMP

## 8. Экспорт / Импорт

- **💾 Экспорт** → сохранить результат в .json
- **📂 Загрузить** → открыть предыдущий скан
- **Вкладка JSON** → raw данные для копирования

```python
import json
with open('netmap-export.json') as f:
    data = json.load(f)
for d in data['devices']:
    print(f"{d['ip']} | {d['mac']} | {d['vendor']}")
```

## 10. Ограничения

- **ARP** — устройство должно быть в ARP-таблице ОС
- **ICMP** — может быть заблокирован файрволом
- **SNMP** — v2c, требует знания community
- **nmap** — для Глубокого режима должен быть установлен отдельно
- **Права** — админ-права не требуются (кроме nmap на Windows для SYN-скана)
